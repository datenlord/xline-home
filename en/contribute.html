Contribute

As an open-source project, Xline cannot grow without the support and participation of contributors from the community. If you would like to contribute to this project, be it Xline's code, documentation, or even our website, we would appreciate your help. We are glad to provide any support along the way.

How to be a contributor?

As soon as a PR(Pull Request) proposed by you is approved and merged, you can become an Xline contributorÔºÅ

Pick an area to contribute

You can choose one of the following areas to contribute to:

CURP Protocol

The CURP distributed consensus algorithm implemented in Rust. Read Contribute to Xlinefor details on how to make contributions to the CURP code base.

Xline Server

The Xline distributed KV storage backend implemented in Rust. Read Contribute to Xline for details on how to make contributions to the CURP code base.

Find an issue to work on

For beginners, we have prepared many suitable tasks for you. You can check out, for example, our Help Wanted issues in the TiKV repository.

See below for some commonly used labels across major repositories listed in

  • bug - Indicates an unexpected problem or unintended behavior; it can be small or big in scope
  • good first issue - An ideal first issue to work on for first-time contributors, with mentoring available
  • help wanted - Indicates that a maintainer wants help on an issue or pull request
  • discussion - Status: Under discussion or needs discussion
  • enhancement - New feature or request
  • question - Further information is requested, or the question is to be answered.

Ask a question

If you need any help to get started, understand the codebase, or make a PR(or anything else really), please feel free to ask us on Gitter.
